Outra coisa também, é que eu passei por este problema pois eu estava tentando criar minha própria linguagem de banco de dados em Linguagem C.. era como se fosse uma outra versão do "SQL" mas com sintaxe e comando diferentes, meu sistema seria programado pelo CMD.. e até consegui desenvolver alguns comandos, para operações de banco de dados, porém eu armazenava as tabelas e bancos no arquivo binário em forma de "Inteiros", eu pegava cada caractere de uma String e convertia para inteiros e só assim eu conseguia armazenar em um arquivo binário. A linguagem se chamaria "PXDB" que seria "PlaXDataBase".. Plax é uma linguagem multi-paradigma interpretada em Java que eu criei para processamento de imagens, inteligência artificial e construção de protótipos de sistemas (Web, Desktop e Mobile), porém as características da linguagem Plax não está completa, eu só desenvolvi variáveis, estruturas de condição, repetição e outras novas estruturas como: o WHERE e WHEN (que é a mesclagem de threads, loops e condicionais), e o DYNAMIC que processa um exemplo de IA de forma "funcional".. ai tive a idéia de interligar esta linguagem feito em Java com um banco de dados próprio feito em C - ai surgiu o Plax Database (PXDB), onde contém uma sintaxe ainda mais simples do que o próprio SQL. 
O PXDB teria algumas características, ex:
  * Cada banco seria um arquivo binário com a extensão ".db"  * As tabelas, campos e valores estariam armazenadas em um arquivo de um Banco.  * As Construções de bancos, tabelas e campos teriam uma sintaxe simples e objetiva.  * O processamento da linguagem PXDB seria feito por um executável em C.  * A programação PXDB poderia ser feita de 2 formas: Interativa (CMD) ou Programada (Plax).  * O executável do PXDB teria comunicação por Socket com um Servidor em Java.  * O servidor em Java seria programado pela linguagem Plax.
